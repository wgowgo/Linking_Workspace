개인정보 보호 정책

마지막 업데이트: 2024년 1월

Linking Workspace는 사용자의 개인정보를 수집하거나 외부 서버로 전송하지 않습니다. 이 정책은 Linking Workspace Chrome 확장 프로그램의 개인정보 처리 방침을 설명합니다.

1. 데이터 저장

Linking Workspace는 모든 데이터를 사용자의 브라우저 로컬 저장소에만 저장합니다:

  • 노트: 사용자가 작성한 모든 노트는 브라우저 로컬 저장소에 저장됩니다.
  • 할 일: 작업 목록 및 완료 상태가 로컬에 저장됩니다.
  • 하이라이트: 웹 페이지에서 저장한 하이라이트 텍스트가 로컬에 저장됩니다.
  • 나중에 보기: 저장한 URL 및 페이지 이름이 로컬에 저장됩니다.
  • 설정: 언어, 테마, 단축키 등 모든 설정이 로컬에 저장됩니다.

중요: 모든 데이터는 사용자의 기기에만 저장되며, 외부 서버나 클라우드로 전송되지 않습니다. 데이터는 Chrome의 로컬 저장소(최대 10MB)에 저장됩니다.

2. 외부 서비스 사용

Linking Workspace는 다음과 같은 외부 서비스를 사용합니다:

  • TinyURL API: URL 단축 기능 사용 시, TinyURL 서비스(https://tinyurl.com)를 사용합니다. 사용자가 명시적으로 "URL 단축" 버튼을 클릭한 경우에만 해당 URL이 TinyURL 서비스로 전송됩니다. TinyURL의 개인정보 보호 정책은 해당 서비스 웹사이트를 참조하시기 바랍니다.

3. 권한 사용

Linking Workspace는 다음 권한을 사용합니다. 각 권한은 확장 프로그램 기능에 필수적입니다:

  • storage: 로컬 데이터 저장 (노트, 할 일, 하이라이트, 북마크 등)
  • activeTab: 현재 탭의 콘텐츠에 접근 (하이라이트 기능 및 페이지 정보 수집)
  • scripting: 콘텐츠 스크립트 주입 (하이라이트, 광고 차단 기능)
  • contextMenus: 우클릭 메뉴 제공 (하이라이트, 나중에 보기 기능)
  • windows: 팝업 윈도우 생성 (나중에 보기 추가 시)
  • notifications: 타이머 알림 (포모도로 타이머 완료 시)
  • host_permissions (tinyurl.com): URL 단축 기능 (사용자 요청 시에만 사용)

4. 데이터 수집 및 공유

Linking Workspace는 사용자의 개인정보를 수집하거나 제3자와 공유하지 않습니다. 모든 데이터는 사용자의 기기에서만 처리되고 저장됩니다.

5. 데이터 삭제

사용자는 언제든지 확장 프로그램을 삭제하여 모든 데이터를 제거할 수 있습니다. 확장 프로그램 삭제 시 Chrome의 로컬 저장소에서 모든 데이터가 함께 삭제됩니다.

6. 보안

모든 데이터는 브라우저의 표준 보안 기능을 통해 보호됩니다. 데이터는 암호화되지 않지만, Chrome의 로컬 저장소에 저장되어 다른 웹사이트나 확장 프로그램에서 접근할 수 없습니다.

7. 정책 변경

이 개인정보 보호 정책은 변경될 수 있습니다. 중요한 변경사항이 있을 경우, 확장 프로그램 업데이트를 통해 알려드리겠습니다.

Privacy Policy

Last Updated: January 2024

Linking Workspace does not collect or transmit user personal information to external servers. This policy explains how the Linking Workspace Chrome extension handles personal information.

1. Data Storage

Linking Workspace stores all data only in the user's browser local storage:

  • Notes: All notes written by users are stored in browser local storage.
  • Tasks: Task lists and completion status are stored locally.
  • Highlights: Highlighted text saved from web pages is stored locally.
  • Read Later: Saved URLs and page names are stored locally.
  • Settings: All settings including language, theme, and shortcuts are stored locally.

Important: All data is stored only on the user's device and is not transmitted to external servers or cloud services. Data is stored in Chrome's local storage (up to 10MB).

2. External Services

Linking Workspace uses the following external service:

  • TinyURL API: When using the URL shortening feature, the TinyURL service (https://tinyurl.com) is used. URLs are sent to the TinyURL service only when the user explicitly clicks the "Shorten URL" button. Please refer to TinyURL's privacy policy on their website.

3. Permission Usage

Linking Workspace uses the following permissions. Each permission is essential for the extension's functionality:

  • storage: Local data storage (notes, tasks, highlights, bookmarks, etc.)
  • activeTab: Access to current tab content (highlight feature and page information collection)
  • scripting: Content script injection (highlight, ad blocking features)
  • contextMenus: Right-click menu (highlight, read later features)
  • windows: Popup window creation (when adding to read later)
  • notifications: Timer notifications (when Pomodoro timer completes)
  • host_permissions (tinyurl.com): URL shortening feature (used only upon user request)

4. Data Collection and Sharing

Linking Workspace does not collect or share user personal information with third parties. All data is processed and stored only on the user's device.

5. Data Deletion

Users can delete all data at any time by uninstalling the extension. When the extension is uninstalled, all data is removed from Chrome's local storage.

6. Security

All data is protected through the browser's standard security features. Data is not encrypted but is stored in Chrome's local storage, making it inaccessible to other websites or extensions.

7. Policy Changes

This privacy policy may be updated. If there are significant changes, we will notify users through extension updates.
